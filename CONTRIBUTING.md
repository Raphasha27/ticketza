# Contributing Guidelines

Thank you for contributing to this Kirov Dynamics repository!

## Development Workflow
1. Fork the repository and check out your feature branch:
   ```bash
   git checkout -b feature/amazing-new-feature
   ```
2. Follow standard coding conventions and enforce lints.
3. Commit with semantic, clear messages (Conventional Commits: `feat:`, `fix:`, `chore:`, `docs:`).
4. Run active tests before submitting pull requests:
   ```bash
   # Python: pytest
   # Node: npm run test
   ```
5. Ensure a **zero-vulnerability footprint** (run dependency auditing).

We appreciate your effort to keep our systems secure and robust!
